# SEPA Position Sizer

SEPA Position Sizer 是一個 SEPA 風格的 Web 倉位試算器，用來在進場前快速檢查「型態停損是否足夠近」以及「本次應該投入多大倉位」。

工具核心不是選股，也不是損益紀錄，而是把 Mark Minervini 風格的低風險入場觀念轉成可重複試算的流程：

```text
輸入型態參數 -> 檢查停損是否合理 -> 計算本次倉位
```

## 適用情境

- 已經透過其他工具或人工研究找到候選股。
- 已經判斷目前可能是 VCP、High Tight Flag 或其他 SEPA 風格入場型態。
- 想在下單前確認型態停損是否超過可承受風險。
- 想根據目前獲利緩衝與風險級別，快速計算本次可投入金額、股數與最大虧損。

## 核心概念

### 滿倉投入百分比

滿倉投入百分比是單一股票在完整倉位時，最多可以佔總資金多少。

例如總資金 1,000,000 元，滿倉投入百分比 25%，代表這檔股票的完整倉位上限是 250,000 元。這是部位曝險限制，不是虧損風險限制。

### 滿倉 1R 風險

滿倉 1R 風險是完整倉位允許承擔的最大虧損金額，可以用總資金比例或手動金額設定。

例如總資金 1,000,000 元，滿倉 1R 風險 1%，代表完整倉位最多允許虧損 10,000 元。

### 絕對停損價

絕對停損價是由滿倉投入金額與滿倉 1R 風險反推的最遠停損位置。

如果本次停損價低於絕對停損價，代表型態停損太遠，這筆交易不符合低風險入場條件。常見處理方式是放棄交易、等待更接近停損的位置，或降低入場價格，而不是硬把停損放寬。

### 本次停損價

本次停損價是根據型態失敗位置決定的停損價，例如 VCP 最後一段收縮低點或突破後不可跌破的位置。

工具會用本次停損價和絕對停損價比較，確認這筆交易是否符合風險邏輯。

### 風險級別

風險級別用來模擬「先小倉試單，做對再加大，連錯就縮小」的倉位策略。

工具會根據獲利緩衝估算本次可用級別，例如 1/4 倉、半倉或滿倉。風險級別不是型態分數，也不是股票評分，而是這次交易允許使用完整倉位的多少比例。

### 參考止盈

參考止盈會以入場價和本次停損價計算每股風險，再推導 1R、2R、3R 價格。這些價格只作為分批停利或檢查報酬風險比的參考。

## 使用流程

1. 輸入總投資資金、已實現損益、未實現浮盈與浮盈折扣百分比。
2. 設定滿倉投入百分比與滿倉 1R 風險。
3. 輸入入場價與本次停損價。
4. 檢查本次停損價是否低於絕對停損價。
5. 查看風險級別、本次風險上限、建議股數、投入金額與最大虧損。
6. 參考 1R、2R、3R 價格規劃分批停利。

設定會自動保存到瀏覽器 localStorage。若要回到預設值，可點擊「重設設定」。

## 線上部署

GitHub Pages 網址：

```text
https://MagicalWater.github.io/sepa-position-sizer/
```

本專案使用 GitHub Actions 自動部署。每次 push 到 `main` 後，workflow 會執行：

```text
npm ci
npm test
npm run build
部署 dist 到 GitHub Pages
```

第一次啟用 GitHub Pages 時，請到 GitHub repo 的設定頁：

```text
Settings -> Pages -> Build and deployment -> Source -> GitHub Actions
```

## 專案定位

這個工具可以和其他 SEPA 研究流程搭配，但不直接負責選股、基本面篩選、RS Rating 排名或交易紀錄。

目前定位：

- `rs-theme-screener`：候選股與主題研究。
- `fundamental-growth-screener`：候選股基本面成長驗證。
- `sepa-position-sizer`：入場前倉位與停損風險試算。

## 風險聲明

本專案只提供倉位試算與風險檢查，不構成任何投資建議、買賣建議或收益保證。實際交易仍需自行判斷市場環境、個股流動性、滑價、跳空風險與個人風險承受能力。
